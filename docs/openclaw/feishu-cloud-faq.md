## 一、使用入门

### Q1：飞书云 Claw 装好后，我第一步该做什么？

先做 3 件事：

1. 在飞书里确认它能正常回复
2. 安装第一批基础 Skills
3. 创建一个最简单的定时提醒

不要先去研究复杂配置。

### Q2：我已经能聊天了，是不是就算配置完成了？

不是。

“能聊天”只说明基础链路通了，不代表它已经开始真正帮你干活。通常还要补：

- Skills
- 定时任务
- 搜索能力
- `USER.md`
- `data/` 目录

## 二、Skills 相关

### Q4：我应该先装哪些 Skills？

推荐第一批：

1. `skill-vetter`
2. `backup`
3. `browser`
4. `shell`
5. `scheduler`
6. `weather`

如果你只想最小化开始，也至少先装：

- `scheduler`
- `weather`
- `brave-search`

### Q5：为什么不建议一开始装很多 Skills？

因为一旦出问题，你很难判断：

- 是哪个 Skill 配坏了
- 是哪个 Skill 缺 Key
- 是不是两个能力叠加后才出问题

先少装，先确认能用，再继续加。

### Q6：装了 Skill 之后，怎么判断它真的在工作？

不要只看“装上了”，要给它一个必须调用该 Skill 的任务。

例如：

- `weather`：问今天天气
- `browser`：让它打开网页并总结
- `shell`：让它检查服务器资源
- `scheduler`：让它 10 分钟后提醒你

### Q7：为什么搜索最新信息时，回答还是旧的？

通常是因为：

1. 没装 `brave-search`
2. 装了但没配 API Key
3. 配了 Key 但没重启网关

## 三、定时任务相关

### Q8：我应该先做什么样的定时任务？

先做最简单的测试提醒：

```text
请帮我创建一个测试定时任务：10 分钟后提醒我“scheduler 已经生效”。
```

跑通后再做每天/每周的周期任务。

### Q9：为什么我创建了定时任务，但没有收到提醒？

优先检查：

1. `scheduler` 是否已安装
2. 网关是否在运行
3. 飞书链路是否正常
4. 你是不是一上来就做了复杂组合任务

### Q10：为什么定时任务触发了，但内容不对？

通常是因为组合依赖不完整。

例如你让它：

- 查天气
- 读 `todo.md`
- 搜索新闻

但实际上：

- `weather` 没装
- `brave-search` 没配 Key
- `todo.md` 文件不存在

### Q11：最值得长期保留的 3 个定时任务是什么？

对大多数用户来说，最值得先保留的是：

1. 每天早上 8 点天气或开工提醒
2. 每个工作日 9 点今日重点提醒
3. 每周五下午 5 点周报提醒

## 四、飞书里的用法

### Q12：在飞书里怎么问，效果会更好？

尽量说清楚：

1. 要做什么
2. 在什么时候做
3. 是否要读文件
4. 最后输出成什么形式

差写法：

```text
帮我弄个提醒
```

好写法：

```text
请帮我设置一个定时任务：每周五下午 5 点提醒我写周报，并按“本周完成、问题、下周计划”的结构提示我整理内容。
```

### Q13：飞书云 Claw 最适合做什么？

最适合做：

- 提醒
- 搜索
- 简单网页检查
- 服务器轻量巡检
- 草稿和摘要整理

不建议一开始就让它承载太复杂、太长链路的自动化。

### Q14：它更适合日常聊天，还是更适合自动化？

如果已经装好并配对成功，它最大的价值通常在“轻量自动化”，而不只是聊天。

## 五、文件和配置

### Q15：我需要 `USER.md` 吗？

需要，建议至少有一个最小版本。

因为它能让飞书云 Claw 更稳定地理解：

- 你默认用什么语言
- 你偏好的输出方式
- 哪些事要先确认

### Q16：我需要 `data/` 目录吗？

如果你后面想做：

- 待办简报
- 周报辅助
- 固定模板输出

那非常建议建。

至少准备：

- `todo.md`
- `weekly-template.md`

### Q17：我一定要去改很多配置文件吗？

不一定。

如果你只是轻度使用，很多事情在飞书里直接说就可以。真正需要手动补的，通常是：

- 搜索类 Skill 的 API Key
- 更明确的 `USER.md`
- `data/` 目录里的长期资料

## 六、故障和维护

### Q18：飞书里突然不回复了，我先做什么？

先做最短排查：

1. 在飞书里发一句最简单的消息
2. 看 `openclaw gateway status`
3. 看最近 100 行日志

不要一上来就重装。

### Q19：升级或改配置后出问题了怎么办？

先回忆最近改了什么，再去看日志。

最常见的问题不是“突然坏了”，而是最近改动引起的。

## 七、如果你只想记住一句话

先把它当成：

> 飞书里的提醒器 + 搜索助手 + 轻量工作流入口

而不是一开始就把它当成全自动系统。

这样最容易用起来，也最不容易挫败。
