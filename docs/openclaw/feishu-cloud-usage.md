# 04A. 飞书云 Claw 安装后使用与配置指南

> **适用对象**：已经完成飞书云 Claw 部署，飞书里已经能正常聊天，但还不知道下一步怎么配置和使用的用户 | **文档版本**：V1.0
> **更新日期**：2026 年 3 月 23 日 | **文档定位**：专门讲“安装完成之后”的实际使用，重点覆盖 Skills、定时任务、常见工作流、日常维护

---

## 先看结论

如果你现在的状态是：

- 云 Claw 已经装好了
- 飞书里已经能和它对话
- 但还不知道接下来该配什么

那正确顺序不是继续折腾部署，而是做这 5 件事：

1. 先确认默认模型稳定可用
2. 装第一批基础 Skills
3. 先做 1 个最简单的定时任务
4. 再补搜索、浏览器、邮箱、GitHub 这类进阶能力
5. 建立自己的“飞书使用习惯”

对大多数用户来说，真正能把云 Claw 用起来，靠的不是“更多安装”，而是“更少但更实用的配置”。

---

## 一、你现在应该先确认什么

先不要继续加功能，先确认下面 4 项：

### 1.1 飞书里能稳定回复

最基本的测试：

- 在飞书里发一句“你好”
- 再发一句稍复杂的任务，比如“帮我总结一下今天要做的三件事”

如果能正常回复，说明飞书接入和模型链路是通的。

### 1.2 Web 控制台还能打开

飞书能用，不代表 Web 控制台一定没问题。建议保留 Web 控制台，后面很多配置和排查更方便。

### 1.3 当前默认模型是你想要的

云端装完后，很多人就一直用镜像默认模型，结果后面发现：

- 太贵
- 太慢
- 中文效果不合适

如果你已经有明确偏好，建议先确认默认模型。

模型说明参考：

- [06-OpenClaw-模型配置-通用说明.md](./model-config.md)

### 1.4 先别一口气装太多 Skill

最容易出问题的路径就是：

- 刚装好
- 一次装十几个 Skills
- 再开好几个自动化任务

这样后面出了问题，你根本不知道是哪一步坏了。

---

## 二、飞书云 Claw 装好后，第一批该配什么

先装“最常用、最稳、最容易立即见效”的 6 个 Skills：

1. `skill-vetter`
2. `backup`
3. `browser`
4. `shell`
5. `scheduler`
6. `weather`

安装方式：

```bash
npx clawhub@latest install skill-vetter
npx clawhub@latest install backup
npx clawhub@latest install browser
npx clawhub@latest install shell
npx clawhub@latest install scheduler
npx clawhub@latest install weather
openclaw gateway restart
```

如果你的云端是带面板的镜像，也可以直接在终端里执行，不需要重新部署。

### 为什么先装这 6 个

因为它们刚好覆盖了最常见的 4 类需求：

- 查信息：`weather`
- 执行操作：`shell`
- 网页处理：`browser`
- 自动化：`scheduler`

`backup` 和 `skill-vetter` 则是稳定性和安全性的底层保障。

---

## 三、装完 Skills 后，怎么验证它们真的生效

不要只看“装上了”，要看“能不能被实际调用”。

你可以直接在飞书里发这些测试消息：

### 3.1 测 `weather`

发：

```text
今天上海天气怎么样？
```

### 3.2 测 `browser`

发：

```text
帮我打开 bilibili.com 看看首页，并告诉我你看到了什么。
```

### 3.3 测 `shell`

发：

```text
帮我看看这台服务器当前磁盘空间还剩多少。
```

### 3.4 测 `scheduler`

发：

```text
请帮我创建一个测试定时任务：5 分钟后提醒我“scheduler 已经生效”。
```

如果这些都能工作，说明你的云 Claw 已经不只是“能聊天”，而是开始具备实际执行能力了。

## 四、飞书云 Claw 最值得先做的 4 个配置

### 4.1 配一个你真正会长期用的默认模型

如果你主要追求性价比，优先考虑：

- `DeepSeek Chat`
- `MiniMax M2.5`
- `通义千问`

如果你主要想要更强效果，再考虑更贵的模型。

不要一开始就同时配 3 到 5 个模型。先有一个稳定主力就够了。

### 4.2 装 `brave-search` 补联网搜索

很多人会在飞书里问“今天发生了什么”“帮我查最新消息”，这类需求如果没有搜索 Skill，模型很容易给旧信息。

安装：

```bash
npx clawhub@latest install brave-search
openclaw gateway restart
```

配置 API Key：

```bash
openclaw config set 'skills.brave-search.config.apiKey' "你的APIKey"
openclaw gateway restart
```

然后在飞书里测试：

```text
帮我搜索今天关于 OpenClaw 的最新资讯，并用中文给我总结。
```

### 4.3 给自己准备一个长期资料目录

如果你后面想做定时简报、日报、待办汇总，建议提前准备：

```text
~/.openclaw/data/
```

常用文件可以放：

- `todo.md`
- `weekly-template.md`
- `contacts.md`
- `writing-style.md`

这样后面定时任务就能直接读取这些资料。

### 4.4 建一个简单的 `USER.md`

即使你现在主要从飞书里用它，也建议创建：

```text
~/.openclaw/USER.md
```

写最少 5 条就够：

```markdown
# 关于我

- 默认使用中文
- 先给结论，再给步骤
- 不确定时直接说明
- 提供命令时优先给可复制版本
- 涉及删除、覆盖、外发时先确认
```

这会明显改善它在飞书里和你对话的稳定性。

## 五、飞书里最实用的 6 类日常用法

### 5.1 让它做“即时助手”

适合随手问：

- 今天北京天气
- 帮我总结这段文字
- 帮我列一个待办清单
- 帮我写一个飞书消息草稿

### 5.2 让它做“搜索助手”

前提：已经配好 `brave-search`

适合问：

- 今天某个行业发生了什么
- 某个产品最近更新了什么
- 帮我对比两家服务商

### 5.3 让它做“浏览器助手”

前提：已经装 `browser`

适合说：

- 帮我打开某网站看看首页内容
- 帮我查某网页上的价格或公告
- 帮我访问一个后台页面并截图

### 5.4 让它做“服务器小管家”

前提：已经装 `shell`

适合说：

- 帮我看看 CPU/内存/磁盘情况
- 帮我看某个目录有哪些文件
- 帮我检查服务是否在运行

### 5.5 让它做“提醒器”

前提：已经装 `scheduler`

适合说：

- 30 分钟后提醒我开会
- 每天早上 8 点给我天气
- 每周一早上 9 点提醒我写周报

### 5.6 让它做“半自动工作流助手”

适合组合使用：

- `scheduler + weather`
- `scheduler + brave-search`
- `scheduler + browser`

这才是飞书云 Claw 最值钱的地方：你平时在飞书里发一句话，它就在后台持续帮你做事。

## 六、定时任务怎么配，最容易上手

很多用户最关心的其实不是 Skill 本身，而是“我能不能让它定时帮我做事”。

答案是可以，但建议按下面顺序来。

### 6.1 第一步：先做一次性测试

在飞书里发：

```text
请帮我创建一个测试定时任务：10 分钟后提醒我“云 Claw 的定时任务已经生效”。
```

如果这个都没跑通，不要急着做每天/每周的周期任务。

### 6.2 第二步：再做固定周期任务

推荐第一个正式任务：

```text
请帮我设置一个定时任务：每天早上 8 点给我发送今天上海的天气预报。
```

如果你已经装了 `weather`，这通常是最稳的起点。

### 6.3 第三步：再做组合任务

例如：

```text
请帮我设置一个定时任务：每天早上 8 点读取 ~/.openclaw/data/todo.md，结合今天北京天气，生成一份简短的早安简报发给我。
```

这个任务同时用到了：

- `scheduler`
- `weather`
- `data/` 资料目录

一旦跑通，你就已经从“聊天机器人”跨进“自动化助手”了。

## 七、推荐你直接复制使用的飞书任务模板

### 7.1 天气提醒

```text
请帮我设置一个定时任务：每天早上 8 点给我发送今天杭州的天气预报。
```

### 7.2 行业资讯简报

前提：装了 `brave-search`

```text
请帮我设置一个定时任务：每天上午 9 点搜索过去 24 小时关于 AI Agent 的重要新闻，筛选最值得关注的 5 条，用中文发给我。
```

### 7.3 待办汇总

前提：你已经准备了 `~/.openclaw/data/todo.md`

```text
请帮我设置一个定时任务：每天工作日上午 9 点读取 ~/.openclaw/data/todo.md，整理成今日待办清单发给我。
```

### 7.4 网页巡检

前提：装了 `browser`

```text
请帮我设置一个定时任务：每天下午 3 点访问指定网页，检查首页公告是否有更新，如有变化就通知我。
```

### 7.5 服务器巡检

前提：装了 `shell`

```text
请帮我设置一个定时任务：每天上午 10 点检查服务器 CPU、内存和磁盘使用情况，如果有明显异常就发消息告诉我。
```

## 八、飞书里怎么提问，效果会更好

很多人不是不会用，而是提问方式太模糊。

在飞书里，建议尽量这样说：

### 好写法

- “帮我搜索今天关于 XX 的最新消息，并给我 5 条摘要”
- “请在 30 分钟后提醒我 XX”
- “帮我检查服务器磁盘剩余空间，并告诉我是否需要清理”
- “请读取 `~/.openclaw/data/todo.md`，整理成优先级清单”

### 差写法

- “你帮我看看”
- “处理一下”
- “弄一下定时任务”

飞书消息本来就比 Web 控制台更短，所以越具体越稳定。

## 九、日常维护时最该做的 5 件事

### 9.1 定期看已安装 Skills

```bash
npx clawhub@latest list
```

### 9.2 更新 Skills

```bash
npx clawhub@latest update --all
openclaw gateway restart
```

### 9.3 出问题先重启网关

```bash
openclaw gateway restart
```

### 9.4 看最近日志

```bash
openclaw gateway logs --lines 100
```

### 9.5 做一次配置备份

最少备份：

```bash
cp ~/.openclaw/openclaw.json ~/openclaw.json.backup
cp ~/.openclaw/USER.md ~/USER.md.backup 2>/dev/null || true
cp -R ~/.openclaw/data ~/openclaw-data.backup 2>/dev/null || true
```

## 十、最常见的 8 个使用误区

### 10.1 以为能聊天就等于配置完成

不是。能聊天只是起点，真正的价值在 Skills 和自动化。

### 10.2 一口气装太多 Skills

很容易把排查难度抬高。

### 10.3 一上来就做复杂定时任务

先做 10 分钟后的测试提醒，再做每天/每周任务。

### 10.4 没给搜索类 Skill 配 Key

这样它回答“最新信息”时很可能还是旧内容。

### 10.5 不准备 `data/` 目录

这样后面想做日报、待办汇总时会很乱。

### 10.6 不写 `USER.md`

这样它每次都像“重新认识你”。

### 10.7 把所有问题都放到飞书里排查

飞书适合日常使用，复杂问题还是 Web 控制台和终端更清楚。

### 10.8 忘了重启网关

很多 Skill 和配置改完后都要：

```bash
openclaw gateway restart
```

---

## 十一、如果你现在只想要一个“最实用配置方案”

直接照这个顺序做：

1. 安装 `skill-vetter`、`backup`、`browser`、`shell`、`scheduler`、`weather`
2. 配好 `brave-search`
3. 建一个最简单的 `USER.md`
4. 建一个 `~/.openclaw/data/todo.md`
5. 在飞书里创建一个“明早 8 点天气提醒”
6. 再创建一个“读取 todo.md 的早安简报”

做到这里，你的飞书云 Claw 基本就进入“真正能帮你干活”的阶段了。
