# 04B. 飞书云 Claw 定时任务模板大全

> **适用对象**：已经能在飞书里使用云 Claw，想直接复制现成定时任务模板的用户 | **文档版本**：V1.0
> **更新日期**：2026 年 3 月 23 日 | **文档定位**：收集适合飞书云 Claw 的定时任务模板，强调“复制即用、先小后大”

---

## 先看结论

飞书云 Claw 的定时任务不要一上来就做复杂工作流。

最稳的顺序是：

1. 先做一次性提醒
2. 再做每天固定任务
3. 再做组合型任务
4. 最后再做巡检和半自动流程

只要你已经装了 `scheduler`，很多场景都能直接开始。

---

## 一、开始前先确认什么

至少确认这两项：

1. `scheduler` 已安装
2. 飞书里已经能正常收到普通回复

先做一个最小测试：

```text
请帮我创建一个测试定时任务：10 分钟后提醒我“飞书云 Claw 的 scheduler 已经生效”。
```

这个跑通后，再做下面的长期任务。

## 五、适合“组合能力”的高级模板

这些模板比普通提醒更有价值，因为它们会读取资料、查询信息或做巡检。

### 5.1 天气 + 待办早安简报

前提：`scheduler` + `weather`，并准备 `~/.openclaw/data/todo.md`

```text
请帮我设置一个定时任务：每天早上 8 点读取 ~/.openclaw/data/todo.md，结合今天北京天气，生成一份简短早安简报发给我。
```

### 5.2 行业新闻简报

前提：`scheduler` + `brave-search`

```text
请帮我设置一个定时任务：每天上午 9 点搜索最近 24 小时关于 AI Agent 的重要新闻，筛选最值得关注的 5 条，用中文发给我。
```

### 5.3 网页公告巡检

前提：`scheduler` + `browser`

```text
请帮我设置一个定时任务：每天下午 3 点访问指定网页，检查首页公告是否有更新，如有变化就通知我。
```

### 5.4 服务器健康检查

前提：`scheduler` + `shell`

```text
请帮我设置一个定时任务：每天上午 10 点检查服务器 CPU、内存和磁盘使用情况，如果有明显异常就发消息告诉我。
```

### 5.5 邮件巡检

前提：`scheduler` + 邮件 Skill

```text
请帮我设置一个定时任务：每隔 2 小时检查一次新邮件，如果有来自重要联系人且需要回复的邮件，就提醒我。
```

## 六、怎么把任务写得更稳

建议你在飞书里发任务时尽量包含这 4 个要素：

1. 什么时候执行
2. 要做什么
3. 用哪些资料
4. 最后发给谁/怎么输出

例如：

差写法：

```text
帮我弄个早报
```

好写法：

```text
请帮我设置一个定时任务：每天早上 8 点读取 ~/.openclaw/data/todo.md，结合今天上海天气，整理成一份不超过 200 字的早安简报发给我。
```

---

## 七、推荐的任务升级顺序

不要跳级，建议按这个顺序来：

1. 一次性提醒
2. 每天固定提醒
3. 天气类任务
4. 读取 `data/` 文件的任务
5. 搜索类任务
6. 浏览器巡检
7. 服务器巡检

这样每一步出了问题都容易回溯。

---

## 八、常见问题

### Q1：为什么任务创建了，但没收到消息

先检查：

- `scheduler` 是否已安装
- 飞书渠道是否仍正常
- 网关是否还在运行

### Q2：为什么组合任务不稳定

通常是因为：

- 对应 Skill 没装全
- `data/` 文件还没准备
- 任务描述太模糊

### Q3：应该先做哪些任务

优先做：

- 天气提醒
- 会议提醒
- 周报提醒
- 待办简报

这些成功率最高，也最容易感受到价值。
